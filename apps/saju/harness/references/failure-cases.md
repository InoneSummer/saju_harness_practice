# Saju Failure Cases

## 1. Purpose
이 문서는 `saju` 앱에서 자주 틀리기 쉬운 실패 패턴을 모아 둔 참고 문서다.
코더와 테스터는 구현 전후에 이 문서를 빠르게 확인한다.

## 2. Calculation Failures
- 년주, 월주, 일주를 직접 계산해서 기준 라이브러리와 어긋나는 경우
- 시주 조견표 인덱싱이 틀리는 경우
- 시간 경계 처리 실수로 자시가 어긋나는 경우
- 오행 합계가 8이 아닌 경우

## 3. Input Failures
- 존재하지 않는 날짜가 통과하는 경우
- 지원 범위 밖 입력이 통과하는 경우
- 빠진 입력이 조용히 처리되는 경우

## 4. Interpretation Failures
- 해석 분기에서 빠진 경우가 있는 경우
- 오행 0개 케이스가 누락되는 경우
- 해석 텍스트에 입력값이 섞이는 경우

## 5. Security-Linked Failures
- 오류 메시지에 입력값이 과하게 노출되는 경우
- 세션 간 이전 입력값이 다시 보이는 경우
- 디버그 코드가 남아 입력값을 출력하는 경우

## 6. Use
- 구현 전에 현재 작업과 관련된 실패 패턴을 먼저 본다.
- 테스트 설계 전에 빠진 경계 조건이 없는지 확인한다.
- 같은 실패가 반복되면 ongoing plan의 `root_cause`와 연결한다.
