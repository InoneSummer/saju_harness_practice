# Saju Security

## 1. Purpose
이 문서는 `saju` 앱에서 보호해야 할 데이터와 금지 행동을 정의한다.
repository-level security를 기본선으로 두고, 이 앱의 입력과 결과 흐름을 더 구체적으로 다룬다.

## 2. Protected Data
- 생년
- 월
- 일
- 시
- 위 값을 조합해 원래 입력을 추정할 수 있는 모든 문자열

## 3. Allowed Output
- 사주 팔자 결과
- 오행 분포
- 규칙 기반 해석 텍스트
- 입력값을 포함하지 않는 일반 오류 메시지

## 4. Forbidden
- 입력값을 로그, 파일, DB, URL에 남기기
- 입력값을 외부 API나 외부 저장소로 보내기
- 입력값을 디버그 출력에 쓰기
- 다른 세션에서 이전 입력값이 보이게 만들기

## 5. Data Rule
- 입력값은 계산 중 메모리 안에서만 사용한다.
- 입력값은 함수 인자와 반환값 범위 안에서만 흐르게 한다.
- 계산이 끝난 뒤에는 결과만 화면에 남긴다.

## 6. Share Rule
- 공유 링크에 입력값이 포함되면 안 된다.
- 공유 상태에서도 세션 간 입력값이 섞이면 안 된다.
- 사용자에게 저장되지 않음을 안내해야 한다.

## 7. Verification
- 금지 패턴은 자동 검사로 확인한다.
- 데이터 흐름은 리뷰 단계에서 확인한다.
- 보안 의심 사항이 남으면 완료로 표시하지 않는다.
