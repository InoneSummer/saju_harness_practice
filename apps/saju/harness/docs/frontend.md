# Saju Frontend Guide

## 1. Purpose
이 문서는 `saju` 앱의 UI를 어떤 컴포넌트와 흐름으로 구현할지 정의한다.
design 문서의 방향을 실제 화면 구성 규칙으로 내리는 역할을 한다.

## 2. UI Structure
- 입력 영역 (년/월/일/시 + 성별)
- 실행 버튼
- 오류 메시지 영역
- 결과 영역
- 푸터 안내 영역

## 3. Input Rule
- 년, 월, 일, 시, 성별은 선택형 입력으로 둔다.
- 입력은 한 번에 모두 확인할 수 있어야 한다.
- 모바일 표준 화면에서도 입력 순서가 깨지지 않아야 한다.

## 4. Result Rule
- 결과는 팔자, 오행, 해석 순서로 보여준다.
- 오류가 있으면 결과보다 오류를 먼저 보여준다.
- 결과 텍스트는 입력값을 다시 노출하지 않는 방향으로 유지한다.

## 5. Component Rule
- 기본 UI는 Gradio 기본 컴포넌트 안에서 해결한다.
- 결과 카드는 HTML이나 기본 출력 컴포넌트로 표현한다.
- 차트는 오행 개수만 보여주는 단순 구조로 유지한다.
- 푸터에는 보안 안내와 신뢰성 안내를 함께 둔다.

> **Note:** Gradio 6.x 이후부터는 `css` 속성을 `Blocks` 생성자가 아닌
> `launch()` 메서드로 넘겨야 한다.  `app.build_interface` 자체는 스타일을
> 반환 객체에 붙ける만 하고, `app.main()`이 실제로 `demo.launch(css=...)`
>를 호출한다.

## 6. Interaction Rule
- 분석은 사용자가 버튼을 눌렀을 때만 실행한다.
- 잘못된 입력은 즉시 실패로 보여준다.  빈 성별도 오류로 처리된다.
- 성공 시 결과 영역을 갱신하고, 실패 시 오류 상태를 남긴다.

## 7. Out of Scope
- 불필요한 애니메이션
- 복잡한 클라이언트 상태 관리
- JavaScript 중심 인터랙션
