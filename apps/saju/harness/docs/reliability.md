# Saju Reliability

## 1. Purpose
이 문서는 `saju` 앱의 도메인 검증 기준을 정의한다.
repository-level reliability를 기본선으로 두고, 사주 계산과 표시 결과를 어떻게 검증할지 정한다.

## 2. Source of Truth
- 년주, 월주, 일주: `korean-lunar-calendar`
- 시주: 앱 내부 조견표
- 오행 분포: 상수 테이블 기반 매핑
- 해석 텍스트: 사전 정의된 템플릿

## 3. Supported Range
- 입력 연도 범위: 1900~2050
- 검증 기준 날짜 범위: 1900-01-01 ~ 2050-12-31
- 시 입력 범위: 자시~해시의 12개 고정 구간
## 4. Required Checks
- 사주 팔자 계산이 기준 결과와 맞아야 한다.
- 오행 개수 합계가 항상 8이어야 한다.
- 해석 텍스트는 정해진 분기 안에서만 나와야 한다.
- 잘못된 입력은 조용히 통과하지 않고 실패로 드러나야 한다.

## 5. Minimum Test Set
- 고정 샘플 테스트
- 경계 입력 테스트
- 범위 밖 입력 테스트
- 오행 합계 테스트
- 해석 분기 테스트

## 6. Edge Cases
- 자시 경계
- 연초와 연말 경계
- 윤달 관련 날짜
- 지원 범위 하한과 상한
- 절입조정, 서머타임, 야자시처럼 문서로만 제외한 범위

## 7. Failure Rule
- 실패는 기대값과 실제값 차이로 남긴다.
- 같은 원인의 실패가 반복되면 ongoing plan의 `root_cause`를 갱신한다.
- tester 판정은 repository-level verdict 규칙을 따른다.

## 8. Done for This App
- 핵심 계산 결과가 테스트로 검증된다.
- 제외한 범위가 문서에 적혀 있다.
- 실패가 기록 없이 사라지지 않는다.
